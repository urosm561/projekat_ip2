RepeatsPlus program finds maximal repeats in input sequences.

Option Fasta file:

Fasta file specifies input file that is expected to be in Fasta format.

Option Sequence type:

Sequence type specifies the input sequence type (DNA, RNA or Protein).

Option Minimal repeat length:

Minimal length specifies the minimal length of the repeats that are extracted from input file.

Option Repeat type:

RepeatType specifies the type of repeats that are extracted. Possible types are direct non-complementary, indirect non-complementary, direct complementary and indirect complementary.

Option MaxGap:

MaxGap specifies the maximal gap between two components that make the repeat. If maxgap is zero, tandem repeats are found. If not specified all repeats are found (not depending on their gap).

Option Ambigious letters:

Ambigious letters option specifies that if there are ambigious letters in input sequence, repeats are found considering all possibilities for ambigious letters.

Option motif:

Motif specifies a mask that is used to find all repeats that satisfy given motif mask. In defining motif mask next rules are used:

  • string of characters drawn from the alphabet denotes a sequence of the corresponding characters;
  • character '.' matches any character from alphabet;
  • characters in brackets denote any one caracter from brackets;
  • characters in brackets starting with ^ denote any one character from alphabet that is not in the brackets.

If option mismatch is checked then found repeats may have a mismatch. In this case statistical filtering is not provided.

Option P value:

P value specifies number that is used as p value in case of statistic filtering.

Option Custom complement mapping file:

A custom complement mapping file can be used. Specifies the file with the letters and their complements . The file should be a text file with an even number of letters. The first two letters form a pair with the first letter having the second letter as a complement, then the second two letters form another pair, etc.

Option Protein wildcards:

Different protein wildcards can be used as: Aliphatic, Sulphur, Tiny…

Option Treat all sequence in Fasta file as single sequence:

If input file contains multiple Fasta files they can be treated as a single input file. If this option is not chosen and input file contains multiple Fasta files they will be treated one by one.

Option Exclude letter N segments:

The letter N in genomic sequences is used to specify an unknown nucleic acid. Large islands of N's typically lead to an explosion of found pairs. This option instructs program to skip over any island of number or more consecutive N's if number is specified. Excludes every consecutive number or more letters N from repeat searching.  If  0 is specified than all N will be excluded.

Option Output file name:

Specifies name of input file.

Option ODBC connection string:

Specifies ODBC connection string used to connect to database. Results can be written directly either to DB2 or to other relational database with an ODBC driver installed. A sample database schema can be found in dbschema.txt.
